## 🔒 安全问题详情

### 1. SQL 注入漏洞
**文件**: `Sills/db_cli.py`, `Sills/db_vendor.py`
**风险等级**: 🔴 高
**描述**: 使用 f-string 拼接 SQL 语句，存在注入风险
```python
sql = f"INSERT INTO uni_cli ({columns}) VALUES ({placeholders})"
```
**建议**: 使用参数化查询或白名单验证列名

### 2. MD5 密码哈希
**文件**: `Sills/db_emp.py`
**风险等级**: 🔴 高
**描述**: 使用 MD5 进行密码哈希，已被证明不安全
**建议**: 升级为 bcrypt 或 argon2

### 3. 硬编码后门账户
**文件**: `Sills/base.py`
**风险等级**: 🔴 高
**描述**: 初始化时创建固定后门账户 (Admin/uni519)
**建议**: 移除或改为首次启动时由用户设置

### 4. 默认密码
**文件**: `Sills/db_emp.py`
**风险等级**: 🟡 中
**描述**: 新员工默认密码 '12345'
**建议**: 要求首次登录时强制修改密码

### 开始一个任务
```bash
git checkout main
git pull origin main
git checkout -b fix/sql-injection
```

### 完成一个任务
```bash
git add .
git commit -m "fix: 修复 SQL 注入漏洞 2026-03-17 14:30"
git fetch origin
git rebase origin/main
git push origin fix/sql-injection
# GitHub 上发 PR → 合并到 main → 删除分支
```

### 状态说明
| 图标 | 含义 |
|---|---|
| 🔲 待开始 | 还未创建分支 |

| 👀 待审查 | PR 已提交，等待合并 |
